# 3.2 - ETC - ServeurDocumentation_BookStack

Vous devez configurer un serveur de documentation BookStack avec Docker sur Ubuntu 24.04.

Le serveur doit utiliser une architecture multi-conteneurs avec réseau Docker personnalisé incluant BookStack, MySQL, Nginx (proxy inverse) et phpMyAdmin.

## Configuration spécifique du serveur

Voir [1.1 - Contexte - Directives générales.md](1.1%20-%20Contexte%20-%20Directives%20g%C3%A9n%C3%A9rales.md)

- Configurer le serveur selon votre domaine
- Utiliser un réseau Docker bridge personnalisé
- Isoler la base de données (aucun port exposé à l'extérieur)
- Modifier les mots de passes d'origine des différents services
  - MySQL
  - BookStack
  - PhpMyAdmin 

## Service à installer
- Docker
- Docker Compose
- Nginx (conteneur)
- MySQL (conteneur)
- BookStack (conteneur)
- phpMyAdmin (conteneur)

*La liste de ces services n'est pas exhaustive, il peut y avoir des dépendances.*

## Tâches après installation
- Configurer les variables d'environnement (.env)
- Créer le réseau Docker personnalisé
- Configurer le proxy inverse Nginx
- Vérifier la persistance des données (volumes)
- Créer du contenu de test dans BookStack :
  - Un livre nommé "Documentation Technique"
  - Un chapitre avec au moins 2 pages
  - Upload d'une image ou fichier

## Validations et tests
- Accès à BookStack via le navigateur
- Accès à phpMyAdmin via le proxy Nginx
- Vérification de l'isolation réseau de MySQL
- Tests de persistance des données après redémarrage
- Inspection du réseau Docker et des conteneurs
- Vérification de la résolution DNS inter-conteneurs

*Liste non exhaustive, d'autres tests peuvent être nécessaire*
